자바스크립트의 함수

함수는 프로그래밍 언어에서 가장 중요한 실행을 담당한다. 함수는 실행의 그룹화된 가장 작은 단위로 보통은 사용되나 함수지향 언어인 자바스크립트에서는 클래스처럼 사용되기도 한다.

자바스크립트의 실행은 순서를 따르지만 함수는 어디에 선언해도 참조할 수 있다.

자바스크립트는 같은 이름의 함수(파라미터가 갯수가 다른)를 여러개 가질수 없다(오버로딩x).떄문에 함수를 선언할때 상수에 익명함수를 참조하는 형식이 좋다. 예) const a = function(){}

기본 선언법:
function name(parameter){ 실행 내용
return 값을 반환하고 함수를 종료함.
}

익명함수란 : 인스턴스 즉시 쓰고 사라질 함수로 보통은 매개변수로 쓰일 때 가볍게 선언하려 만들어진다. 예) document.onclick= function(){} 익명함수.이벤트시 발생하고 사라진다. const 로 선언한 함수는 이름을 가짐.

ES6 에서 추가된 arrow function : const a=(p)=>{return};
ex) document.onlclick=()=>{};
document.addEventListner("onlclick",()=>{});

const a= function(a,b=default): 함수의 매개변수는 default 값을 가질 수 있다. 이때 a는 function(a),function(a,b) 둘다 선언한것과 같으니 오버로딩과 같다.(ES6에 추가.)

const a=function(...args){}; a(1,2,3,4,5...) ...args는 함수의 매개변수 개수를 제한하지 않는다. 이때 args 는 배열 타입이다.
